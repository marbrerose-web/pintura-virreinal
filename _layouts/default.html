It owns the document and the shared chrome that should appear site-wide. What it renders — the full document skeleton: a that wires up the CSS cascade-layer order, Bootstrap 5 (loaded into the `third-party` layer so Telar's own rules win by layer precedence regardless of source order), theme-driven Google Fonts, the compiled Telar stylesheet, optional KaTeX, and jekyll-seo-tag meta; then a with the site header, the page's {{ content }} inside
, the footer, and the shared glossary/share panels, closing with Bootstrap's JS bundle and Telar's own scripts. Key context — `site.title`/`page.title` for the ; `site.baseurl`, exposed to client JS via a <meta> tag so scripts (e.g. objects-filter.js) can build correct fetch paths under a non-empty baseurl rather than guessing from the URL; the active theme under `site.data.themes[site.telar_theme]`, from which the heading/body font families are derived; and `site.telar_language` (falling back to English) for the language data bundle. Non-obvious behaviour — fonts are chosen dynamically from the theme rather than hard-coded, so swapping themes re-wires the Google Fonts request. KaTeX is only pulled in when a page opts into LaTeX.
